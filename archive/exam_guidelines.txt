Lab 5 for Part A.

Your system must be executable and demonstrably functional. Hardcoded outputs, mocked

3. Modular and Clean Code Structure 
4. Proper Naming Conventions

A proper technical report must accompany your submission. Poor documentation, even with 
working code, will affect marks.

• Missing files or incomplete components will receive zero for that section.

Overly complex but unstable systems will not score higher than clean, well-structured, fully 
functional implementations.

Part A: 
You are required to continue the same domain/project you have been developing in previous labs. 
You must implement all technical requirements exactly as defined in the official Lab Manual. 
Failure to follow lab specifications will result in deduction of marks.          (40 Marks) 
Task 1:                                                  [CLO-1/ GA-4/P4]               (Marks: 10)                                
Design and implement a complete Retrieval -Augmented Generation (RAG) pipeline for your 
project domain.  
All requirements must strictly follow the specifications defined in Lab 2: Knowledge Engineering & 
Domain Grounding. 
Task 2:                                                     [CLO-1/GA-4/P4]              (Marks: 10)      
Design and implement the complete reasoning workflow of your agent using LangGraph. 
Your system must: 
• Define an appropriate graph state structure. 
• Implement an agent node responsible for reasoning. 
• Implement a tool execution node. 
• Develop project-specific tools using proper decorators and input validation. 
• Implement conditional routing logic to control execution flow. 
Your workflow must demonstrate a functioning ReAct loop (Reason + Act). 
All implementations must comply fully with Lab 3: The Reasoning Loop requirements 
Task 3:                                                    [CLO-2/GA-4/P4]              (Marks: 10)      
Extend your system into multi-agent architecture using LangGraph. 
You must implement the following: 
• Define at least two specialized agent personas. 
• Restrict tool access according to agent roles. 
• Implement proper state handover between agents. 
• Design routing logic to manage inter-agent communication. 
• Demonstrate collaborative task execution requiring cooperation between agents. 
Your output must clearly reflect distinct agent responsibilities and transitions. 
All work must comply with Lab 4: Multi-Agent Orchestration requirements.

Task 4:                                                    [CLO-2/GA-4/P4]              (Marks: 10) 
Implement Persistent Memory and Human-in-the-Loop (HITL). 
You must: 
• Integrate persistent state management using a check pointer. 
• Enable session recovery using thread identifiers. 
• Identify a high-risk action tool within your project. 
• Implement a safety interruption mechanism before execution of that tool. 
• Allow human approval or cancellation before proceeding. 
• Demonstrate that a human can edit the agent’s proposed action before execution. 
All work must strictly follow the technical specifications provided in 
Lab 5: State Management & Human-in-the-Loop. 
 
 
Part B: 
You are required to implement a standalone Model Context Protocol (MCP) pipeline as a 
separate task. 
This task must NOT be integrated into your Part A project. 
You must design and demonstrate MCP in an independent use-case of your choice. 
You must strictly follow the MCP design principles. 
Failure to properly demonstrate MCP structure, context separation, and tool exposure will result 
in deduction of marks.                                        Marks:30 
Task 1:                                                    [CLO-1/ GA-4/P4]            (Marks: 10) 
Design and implement an MCP Server that exposes structured tools to a client model. 
You must: 
• Create a standalone MCP server. 
• Define at least two callable tools inside the server. 
• Clearly define structured input and output schemas. 
• Ensure tools are exposed via proper MCP protocol specification. 
• Demonstrate that the model accesses tools through MCP rather than direct function 
calls. 
The implementation must clearly separate: 
• Model 
• Context 
• Tools 
• Execution layer
