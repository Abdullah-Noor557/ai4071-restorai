REACT LOOP EXPLAINED
═══════════════════════════════════════════════════════════════════

The ReAct (Reason + Act) pattern:

1. REASON: Agent thinks about what to do
   "I need to find water ring removal techniques"

2. ACT: Agent calls a tool
   search_restoration_knowledge(query="water ring removal")

3. OBSERVE: Agent receives tool result
   "Iron and cloth method: Place cloth over ring..."

4. REASON: Agent thinks with new information
   "Good, now I need safety information"

5. ACT: Agent calls another tool
   search_restoration_knowledge(query="safety", safety_only=True)

6. OBSERVE: Agent receives safety info
   "Work in ventilated area, no high heat..."

7. REASON: Agent has enough information
   "I can now create the restoration plan"

8. FINAL ANSWER: Agent provides complete plan
   (No more tool calls → Router sends to END)

The conditional router (should_continue function) controls this loop:
- If last_message has tool_calls → route to "tools"
- If no tool_calls → route to "end"
