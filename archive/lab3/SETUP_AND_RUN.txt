═══════════════════════════════════════════════════════════════════
WHAT EACH FILE DOES
═══════════════════════════════════════════════════════════════════

tools.py (617 lines)
--------------------
Defines 3 tools with Pydantic validation:

1. analyze_furniture_image
   - Uses Google Gemini Vision API
   - Input: image_path, analysis_focus
   - Output: JSON with material, damage, finish

2. search_restoration_knowledge (GROUNDING TOOL)
   - Queries Lab 2's ChromaDB vector database
   - Input: query, content_filter, safety_only, n_results
   - Output: JSON with ranked results and metadata
   - Supports metadata filtering from Lab 2

3. search_web_for_products
   - Searches for product availability and prices
   - Input: query, focus
   - Output: JSON with products, retailers, prices

graph.py (665 lines)
--------------------
LangGraph state machine implementation:

- AgentState (TypedDict): State schema
- create_agent_node(): Factory for agent nodes
- should_continue(): Conditional router (ReAct loop)
- create_simple_restorai_graph(): Compiles the graph

Graph structure:
  START → Agent ←→ Tools → END
  
ReAct loop:
  Agent reasons → calls tool → tool executes → result to agent → repeat

main.py (320 lines)
-------------------
Execution script with multiple modes:

- check: Verify environment setup
- demo: Run simulated workflow
- test: Automated testing
- interactive: Process real images

run_demo.py (NEW - Simple version)
-----------------------------------
Simplified demo runner without Unicode issues.
Best for Windows PowerShell testing.
